هدف ما باید یادگیری فلسفه دیزاین پترن ها باشه و نه implementation آنها !!!

فلسفه یک دیزاین پترن یعنی هدف و مکان استفاده ان و نه implementation ان !!! زیرا implemetation میتونه با توجه به مسعله متفاوت باشه و با توجه به خلاقیت برنامه نویس تغییر کنه !!!

انواع design pattern :

  1. creational:
    الگو های این دسته تمرکزشان بر روی ایجاد instance از روی کلاس ها و build کردن object ها هست !!!
  2. structural:
    design pattern های این دسته تمرکزشان بر روی ترکیب موجودیت ها با یکدیگر برای اهداف مختلف میباشد !!!
  3. behavioral:
    design pattern های این دسته تمرکزشان بر روی تعریف چگونه بودن رفتار یک موجودیت با دیگر موجودیت ها برای یک هدف خاص و یا حل یک مشکل خاص
