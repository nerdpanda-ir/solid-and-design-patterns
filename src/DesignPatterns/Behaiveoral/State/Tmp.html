    هدف این design pattern

    این design pattern در جایی استفاده میشود که در یک class و یا object متد هایی برای manipulate کردن یک state و یا property وجود داشته باشد

    مثلا در سیستم ، کلاسی میخواهیم ایجاد کنیم که دارای یک state و یا property باشد و ما نیاز داریم در ان کلاس method هایی را برای انجام عملیات و manipulate بر روی ان state و یا property ایجاد کنیم اما انجام این کار باعث میشود method های نامربوطی با توجه به وظیفه ی class مورد نظرما به آن اضافه شود و آن class دیگر single responsible نباشد و اصل تک مسعولیتی خود را از دست بدهد همچنین با اضافه شدن method های نامربوط به آن class باعث میشود method های زیادی در ان class وجود داشته باشد و کد class ما پیچیده ، شلوغ ، کثیف شود و همچنین class ما پیچیده میشود و کار با ان class برای برنامه نویس client پیچیده ،‌سخت و دشوار شود !!! همچنین با زیاد شدن method های ان class باعث میشود class مورد نظر ما پیچیده شود و ممکن است کدهای آن class شلوغ ، پیچیده و کثیف شود و همچنین کار با ان class برای برنامه نویس client ما پیچیده ، ‌سخت و دشوار شود !!!

    برای حل این مشکلات میتوانیم از دیزاین پترن State استفاده کنیم به این صورت که این design pattern ما را مجبور میکند

    بواسطه دیزاین پترن state دیگر نیاز نیست method های نامربوطی که برای انجام عملیات و یا manipulate بر روی
