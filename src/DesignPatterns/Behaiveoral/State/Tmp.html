• State:

    هدف این design pattern تبدیل state و یا property های یک کلاس به object های مجزا و همچنین انتقال method هایی که بر روی ان state و یا property عملیات انجام میدهند و یا آنرا manipulate میکنند ، ‌از آن class نهایی به ان Object میباشد !!!

    برای حل این مشکلات میتوانیم از دیزاین پترن State استفاده کنیم به این صورت که این design pattern ما را مجبور میکند تا ان state و یا آن property را یک object در نظر بگیریم و method هایی که در آن class وظیفه انجام عملیات و یا manipulate آنرا بر عهده داشتند را در ان آبجکت Encapsulate کنیم !!!!

    بواسطه دیزاین پترن state دیگر نیاز نیست method های نامربوطی که برای انجام عملیات و یا manipulate بر روی
