هدف این دیزاین پترن Encapsulate متد هایی که در یک class بر روی object های آن class یک پردازش و یا عملیات انجام میدهند و سپس ان object را تغییر میدهند و یا manipulate میکنند در class های مجزا میباشد !!!

این Design Pattern در جایی استفاده میشود که یک class دارای method هایی باشد که آن method ها بر روی object هایش یک پردازش و یا عملیات انجام دهد و سپس ان object را تغییر بدهد و یا آن را manipulate بکنند !!!

مثلا در سیستم یک object داریم که برنامه نویس client ما بواسطه method هایی که درون آن object وجود دارد بر روی ان object یک پردازش و یا عملیات انجام میدهد و سپس آن دستکاری میشود و یا آن object را تغییر داده میشود اما وجود چنین method در class آن object که بر روی آن object یک پردازش و یا عملیات انجام میدهند و سپس انرا دستکاری میکنند و یا آنرا تغییر میدهند باعث شده تا مثلا در سیستم class ای داریم که دارای method هایی میباشد که هر یک از instance های ان class میتوانند بواسطه ان method ها ، بر روی خودشان عملیات و یا پردازشی را انجام دهد و یا خودشان را با بواسطه از ان method ها ، تغییر بدهند و یا دستکاری کنند اما وجود چنین method هایی در ان class باعث میشود تعداد method های آن class بیش از حد زیاد شود و ساختار و code آن class نیز پیچیده ، شلوغ و کثیف شود
