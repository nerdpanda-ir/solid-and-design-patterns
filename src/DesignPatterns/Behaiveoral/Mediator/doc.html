• Mediator:

    برای حل این مشکل میتوانیم از دیزاین پترن Mediator استفاده کنیم و communicate و interact کردن آن object ها با یکدیگر را بر عهده این design pattern بسپاریم !!! | به این صورت که این design pattern ما را مجبور میکند تا object هایی را که میخواهند با یکدیگر communicate و interact کنند را به mediator معرفی کنیم و بشناسانیم حالا هر object ای که میخواهد با object های دیگر communicate و interact کند ، میتواند درخواست خود را به mediator بدهد و به دلیل اینکه mediator به همه ی object ها اشراف دارد و همه ی object ها را میشناسد و با همه ی object ها ارتباط دارد آن درخواست را تحویل object مورد نظر میدهد !!!! |

    اینطوری به جای اینکه برنامه نویس Client ما در کدنهایی object ها را به یکدیگر بشناساند و درگیر پیچیدگی ایجاد روابط بین ان object ها شود میتواند هر object را فقط به یونیت Mediator بشناساند و معرفی کند ، حالا هر object ای که بخواهد با یک object دیگر communication و interact کند ، بدون شناخت همدیگر و داشتن رابطه متقابل با هم ، میتواند درخواست خود را به یونیت Mediator بدهد و چون یونیت Mediator همه ی object ها را میشناسد و به همه object ها اشراف دارد آن درخواست را تحویل object مورد نظر میدهد !!!
